Temat projektu:

Tworzenie modeli klasyfikacji wieloetykietowej przez zastosowanie dekompozycji na wiele powizanych zada klasyfikacji jednoetykietowych zgodnie z metod bayesowskiego acucha klasyfikatorw. Porwnanie z algorytmami klasyfikacji wieloetykietowej dostpnymi w rodowisku R lub Python.

(Bayes chain classifier for multi-label classification)

1. Interpretacja tematu projektu

Gwnym celem projektu jest implementacja algorytmu sucego do klasyfikacji przykadw posiadajcych wicej ni jedn etykiet. Kady przykad ma przypisany zestaw binarnych (lub wieloklasowych) znacznikw. Jednym ze sposobw stworzenia modelu umoliwiajcego predykcje na bazie przykadw wieloetykietowych jest dokonanie transformacji tej bazy zgodnie z metod acucha klasyfikatorw. Metoda ta polega na wykorzystaniu koncepcji zwizanej z sekwencyjnym tworzeniem modeli klasyfikacji binarnej, w ktrej to dla kadego kolejnego modelu klasyfikacji dodaje si do zbioru atrybutw predykcje etykiet poprzednich modeli wykorzystanych przy wczeniej rozwaanych etykietach, w poprzednich ogniwach acucha. Dziki temu kady kolejny model estymuje tylko jedn etykiet dla zbioru przykadw. Tym samym wykorzystana jest informacja o zalenoci midzy poszczeglnymi etykietami. Kolejno predykcji dla nastpujcych po sobie etykiet jest istotna i zwykle ma wpyw na kocowy wynik. Dobr kolejnoci ogniw acucha dla predykcji kolejnych etykiet bdzie ustalana na podstawie sieci bayesowskiej utworzonej na podstawie zalenoci midzy poszczeglnymi etykietami. Aby dopeni projekt, wstpnie wybranym klasyfikatorem dla kadego z modeli bdzie gaussowski klasyfikator bayesa.

2.2 Budowa sieci Bayesowskiej

Pierwszym elementem docelowego algorytmu jest budowa sieci bayesowskiej. Sie bayesowska jest tym skadnikiem caego programu, ktry reprezentuje relacje zalenoci pomidzy etykietami. Sie bayesowska to acykliczny graf skierowany, ktrego wzami s numery etykiet, a skierowane krawdzie odzwierciedlaj zaleno przyczynow pomidzy nimi.

Procedura implementacyjna sieci Bayesowskiej korzysta z algorytmu rozpowszechnionego w literaturze pod nazw Chow & Liu. Skada si on z nastpujcych krokw:

  1. Wyznaczenie informacji wzajemnej midzy wszystkimi etykietami:

Funkcja zwraca warto informacji wzajemnej przyjmujc na wejciu prawdopodobiestwa brzegowe 2 etykiet oraz prawdopodobiestwo czci wsplnej obu etykiet.

In[272]:
def mutual_info(puv, pu, pv):
    mi = 0.0
    for (u_val, v_val), p_uv in puv.items():
        p_u = pu[u_val]
        p_v = pv[v_val]
        mi += p_uv * np.log(p_uv / (p_u * p_v))
    return mi
  1. Wyznaczenie drzewa rozpinajcego o maksymalnej sumie wag wykorzystanie algorytmu Kruskala (union, find)

Do implementacji tej czci algorytmu wykorzystano klas DisjointSet. Peni ona rol struktury danych przechowujcej informacj o wierzchokach nalecych do rnych, rozcznych podgrafw. Zawiera metody:

  • find, ktra suy do weryfikacji czy 2 podgrafy s rozczne
  • union, ktra czy 2 rozczne podgrafy w 1

Algorytm Kruskala dziaa tak, e dla kadej pary wierzchokw (etykiet) sprawdza czy s one ju poczone krawdzi (metoda find). Jeli s rozczne to je ze sob czy(metoda union). Algorytm wykonuje si do momentu poczenia wszystkich wierzchokw. Kluczowe w tym algorytmie jest podanie waciwej kolejnoci par wierzchokw na wejciu algorytmu, zgodnej z informacj wzajemn midzy nimi; informacja wzajmna jest wag kadej krawdzi.

import collections

class DisjointSet:

    def __init__(self, *values):
        self.parents = {x: x for x in values}
        self.sizes = {x: 1 for x in values}

    def find(self, x):
        while self.parents[x] != x:
            x, self.parents[x] = self.parents[x], self.parents[self.parents[x]]
        return x

    def union(self, x, y):
        if self.sizes[x] < self.sizes[y]:
            x, y = y, x
        self.parents[y] = x
        self.sizes[x] += self.sizes[y]


def kruskal(vertices, edges):
    ds = DisjointSet(*vertices)
    neighbors = collections.defaultdict(set)

    for u, v in edges:
        if ds.find(u) != ds.find(v):
            neighbors[u].add(v)
            neighbors[v].add(u)
            ds.union(ds.find(u), ds.find(v))

        if len(neighbors) == len(vertices):
            break

    return neighbors
  1. Ustalenie zwrotw krawdzi od wybranego wza pocztkowego.

W tym kroku zadania graf nieskierowany zamienia si na skierowany. W tej implementacji dla sieci bayesowskiej naley poda rwnie wze (etykiet) pocztkow, z ktrej powstanie drzewo osateczna sie. Odczytujc wzy z grafu w kolejnoci topologicznej otrzymamy wejciowe parametry dla acucha klasyfikatorw, mianowicie kolejno etykiet do predykcji.

def orient_tree(neighbors, root, visited):
    #Zwraca pary zalenoci na podstawie roota oraz ssiadow z algorytmu Kruskala
    for neighbor in neighbors[root] - visited:
        yield root, neighbor
        yield from orient_tree(neighbors, root=neighbor, visited={root})

import itertools


def chow_liu(X, root=None):

    # 1 Informacja wzajemna
    marginals = {v: X[v].value_counts(normalize=True) for v in X.columns} #prawdopodobienstwa brzegowe
    edge = collections.namedtuple("edge", ["u", "v", "mi"]) #przydatna struktura
    mutual_info_values = {}
    for u, v in itertools.combinations(sorted(X.columns), 2):
        puv = X.groupby([u, v]).size() / len(X) #prawdopodobienstwo wspolne
        pu = marginals[u]
        pv = marginals[v]
        mutual_info_values[(u, v)] = mutual_info(puv, pu, pv)


    mis = [] #tworzenie listy krotek (u, v, mi) - wezlow polaczonych krawedzia z wagami
    for key, value in mutual_info_values.items():
        u, v = key
        mis.append(edge(u, v, value))

    #sortowanie krawedzi od najbardziej znaczacych
    edges = ((e.u, e.v) for e in sorted(mis, key=lambda e: e.mi, reverse=True))
    edges = list(edges)

    # 2 Maksymalne drzewo rozpinajce
    neighbors = kruskal(vertices=X.columns, edges=edges)

    if root is None:
        root = X.columns[0]

    # 3 zamiana drzewa niezorientowanego na zorientowane
    return list(orient_tree(neighbors, root, visited=set()))

2.3 Implementacja acucha klasyfikatorw

W tym etapie zaimplementowany zosta acuch klasyfikatorw, ktry bdzie jest realizowany przez klas ClassifiersChain. Klasa ta skada si z metody fit() sucej do nauki caych acuchw oraz metody predict() sucej do dawania predykcji klas. Aby utworzy acuch naley poda nazw klasyfikatora sucego do predykcji kolejnych etykiet, a take list, w ktrej to zostanie wskazana kolejno etykiet bdcymi kolejnymi ogniwami acucha.

acuch dziaa tak, e na wejciu dostaje zestaw przykadw oraz kolejne numery etykiet przekazane w licie w odpowiedniej kolejnoci, podyktowanej przez sie bayesowsk. Nastpnie w teje kolejnoci acuch estymuje klas kadej etykiety, przy czym dla kadego kolejnego ogniwa danego acucha wynik poprzedniej predykcji jest traktowany jako dodatkowy atrybut danego przykadu i wiedza ta zostaje przez to wykorzystana do wyliczenia wartoci kolejnej etykiety danego acucha. Innymi sowy dla przykadu skadajcego si z wektora atrybutw X = (x1, x2, x3) i etykiet Y = (y1, y2), pierwszy model stworzony zostanie dla przykadu skadajcego si jedynie z wektora atrybutw X = (x1, x2, x3) i etykiety Y = (y1). Model za spraw klasyfikatora Bayesa dokona predykcji warto y1. W kolejnym podejciu stworzony zostanie drugi model, tym razem ju dla przykadu skadajcego si z wektora atrybutw X = (x1, x2, x3, y1) i etykiety Y = (y2).

from sklearn.base import clone

class ClassifiersChain:
    def __init__(self, classifier, order=None):
        self.cls = classifier  # ustalenie bazowego klasyfikatora
        self.order = order  # ustalenie kolejnoci etykiety do klasyfikacji


    def fit(self, X, y): # metoda trenujca acuch klasyfikatorw
        # ustalenie kolejnoci etykiety do klasyfikacji
        if self.order is None:
            self.order = list(range(y.shape[1]))  # defaultowa kolejnosc

        # lista klasyfikatorw do klasyfikacji poszczeglnych etykiet acucha
        self.models = [clone(self.cls) for _ in self.order]

        self.col_names = y.columns[self.order].tolist()
        X.reset_index(drop=True, inplace=True)
        y_new = y[self.col_names]
        X_joined = pd.concat([X, y_new], axis=1)

        # trenowanie kadego klasyfikatora w acuchu
        for chain_number, model in enumerate(self.models):
            X_ = X_joined.iloc[:, : (X.shape[1] + chain_number)]  # wybranie zestawu atrybutow
            y_ = y_new[self.col_names[chain_number]]  # wybranie etykiety do klasyfikacji
            model.fit(X_, y_)  # trenowanie klasyfikatora na wybranych danych


    def predict(self, X): # metoda dokonujca predykcji nieznanego zbioru danych
        # sprawdzenie czy lista modeli zostaa zainicjalizowana
        if self.models is None:
            raise ValueError("Error. You cannot predict class without training")

        X_joined = X.copy()
        data_types = {col: np.int32 for col in self.col_names}
        pred_chain = pd.DataFrame(columns=data_types.keys()).astype(data_types)

        for chain_number, model in enumerate(self.models):
            if chain_number > 0:
                prev_preds = pred_chain.iloc[
                    :, chain_number - 1
                ]  # wybr poprzednich predykcji
                X_joined = pd.concat([X_joined, prev_preds], axis=1)
            pred = model.predict(X_joined)  # predykcja klasy etykiety
            pred_chain.iloc[:, chain_number] = pred  # zapisanie predykcji

        return pred_chain.astype(np.int32)

W najprostszym wariancie wykorzystania modelu wieloetykietowego, przy tworzeniu acucha klasyfikatorw uwzgldniona zostanie jedynie etykieta rodzica dla danego wza. Zatem algorytm utworzy tyle acuchw ile jest krawdzi w sieci i dla kadej pary wzw stworzy predykcje 2 wartoci. Nastpnie wyniki s czone w jedn tabel.

from sklearn.naive_bayes import GaussianNB

def make_chains(res):
  gnb = GaussianNB() #wybr podstawowego klasyfikatora
  chains = pd.DataFrame(columns=y_test_encoded.columns)

  for i in range(len(res)):
      chain_classifier = ClassifiersChain(gnb, order=list(res[i]))
      chain_classifier.fit(X_train, y_train_encoded)
      y_pred = chain_classifier.predict(X_test)
      if i == 0:
          chains = pd.concat([chains, y_pred])
      else:
          chains = chains.fillna(y_pred)
      chains = chains.astype(int, errors='ignore')

  return chains
